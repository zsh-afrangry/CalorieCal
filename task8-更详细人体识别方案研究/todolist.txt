# Task8 更详细人体识别方案研究 — 实施清单

> 状态：待 Task7 显示优化完成后开始。  
> 目标：研究比 MediaPipe Pose 33 点更细的人体识别、人体表面、人体分割或 3D mesh 方案，判断是否能改善侧身、遮挡、躯干基准不稳定和空间幅度估算。  
> 原则：先研究和小实验，不急于接入实时 /#/demo；必须服务运动强度和热量估算，不为了 3D 而 3D。  

---

## Phase 1: 当前能力边界确认

- [ ] 1.1 记录 MediaPipe Pose 上限
      → 33 个关键点足够做骨架运动量估算
      → 不能直接提供大腿、小腿、躯干表面轮廓或人体 mesh
      → 已发现问题: “双脚合拢”置信度偏高，双腿已经分开时仍可能显示合拢
      → 待研究: 脚距、膝距、肩宽归一化阈值、关键点可见度和时序稳定性是否需要重新校准

- [ ] 1.2 区分显示增强和识别增强
      → Task7: 显示更详细
      → Task8: 识别更详细

## Phase 2: 人体分割方案评估

- [ ] 2.1 评估 MediaPipe segmentation mask
      → 能否提供人体轮廓
      → 能否辅助过滤 depth 背景
      → 是否适合浏览器实时运行

- [ ] 2.2 评估普通人体分割模型
      → 是否能稳定区分人体和背景
      → 是否能辅助 RGB-D depth 采样

## Phase 3: DensePose / 细粒度人体区域

- [ ] 3.1 评估 DensePose
      → 是否能区分躯干、大腿、小腿、手臂等人体区域
      → 是否适合 Python 后端
      → 是否能服务运动幅度和置信度

- [ ] 3.2 判断实时性和部署成本
      → 是否需要 GPU
      → 是否能在当前演示设备上运行
      → 是否只适合离线实验

## Phase 4: SAM 3D Body / Human Mesh Recovery

- [ ] 4.1 离线核对 SAM 3D Body
      → checkpoint、许可证、显存、推理速度
      → 是否能改善侧身、遮挡和躯干参考面

- [ ] 4.2 与 RGB-D 方案比较
      → RGB-D 正面前后方向已可用
      → SAM 3D Body 的价值主要在侧身、遮挡、人体表面和 mesh 展示

## Phase 5: RGB-D 点云和人体区域融合

- [ ] 5.1 评估 depth + 人体分割
      → 用人体 mask 过滤背景 depth
      → 减少采到背景的问题

- [ ] 5.2 评估 depth + 骨架区域采样
      → 不只采 wrist 单点
      → 在肢体附近采样局部区域，提升稳定性

## Phase 6: 结论和后续路线

- [ ] 6.1 给出接入判断
      → 暂缓 / 离线实验 / Python 后端 / 实时演示接入

- [ ] 6.2 记录收益和风险
      → 侧身改善
      → 遮挡改善
      → depth 背景过滤
      → 性能和部署复杂度
