# Task3：手部和上肢运动强度估算

## 本 task 要完成的内容

将 Task1/Task2 的“挥手示例”扩展为更通用的手部和上肢运动强度估算。目标不是识别尽可能多的手势名称，而是即使动作无法精确命名，也能根据上肢关键点运动量给出粗略强度、持续时间、频率、估算 kcal 和置信度。

示例动作可以包括抬手、晃胳膊、伸懒腰、手臂前后移动、简单挥动等。

## 预期结果

1. `/gesture` 从“挥手计数页”升级为“上肢活动强度估算页”。
2. 系统能统计手腕、手掌、肘部或肩部相关关键点的位移、速度、幅度和运动持续时间。
3. 已知动作可以显示辅助标签；未知动作也能按低 / 中 / 高强度给出粗略估算。
4. kcal 仍保持演示可解释，优先说明“基于上肢运动量粗略估算”。
5. 置信度应考虑关键点可见率、运动稳定性、画面残影和是否持续入镜。

## 可能的实现方案和技术栈

1. 前端：Vite + Vue 3 + TypeScript。
2. 视觉识别：MediaPipe Hand Landmarker，必要时叠加 Pose Landmarker 获取肩部、肘部和躯干参考点。
3. 算法：基于关键点轨迹的运动量积分、速度分档、幅度分档和时间窗口统计。
4. 热量估算：从“挥手 MET 档位”扩展为“上肢活动 MET 档位”，先用规则表演示。
5. 后端：暂不强制引入；如果规则表、参数配置或记录管理变复杂，再考虑 FastAPI。

## 边界

1. 不追求动作名称识别准确率。
2. 不做商业级 kcal 精准估算。
3. 不要求识别静态发力或负重强度。
4. 不做全身动作质量纠错，全身能力进入 Task4。
