# Task7：/#/demo 用户体验和界面显示优化

## 本 task 要完成的内容

Task7 重点关注 /#/demo 的用户体验和界面显示，面向商场液晶屏、外场演示和路人快速理解。它不优先新增算法能力，而是把 Task4-Task6 已经形成的运动量、动作证据、RGB-D 状态、强度、kcal 和置信度，以更清晰、更稳定、更适合大屏的方式展示出来。

## 预期结果

1. /#/demo 第一眼能看懂：当前人在动什么、强度如何、估算消耗如何、系统是否可靠。
2. 界面适合商场液晶屏，文字大、层级清晰、无需操作按钮，不依赖用户阅读复杂说明。
3. 摄像头画面、骨架、当前动作、强度、kcal、质量提示之间有明确视觉优先级。
4. depth / RGB-D 状态只作为可读提示出现，不展示过多调试细节。
5. 当识别质量下降时，页面能自然提示“请正对摄像头 / 请进入画面 / 估算置信度下降”。
6. 后续可扩展排行榜、挑战模式、历史累计或活动引导，但 Task7 优先保证基础展示体验。
7. 人体可视化比当前细线骨架更清楚：躯干、大腿、小腿等部位在显示上更像身体结构，而不是只有一条线。

## 可能的实现方案和技术栈

1. 前端：继续使用 Vue 3 + TypeScript + 当前 CSS。
2. 页面：只改 /#/demo，不大改 /#/pose 和 /#/gesture。
3. 展示：左侧保持摄像头和骨架为核心，右侧或底部展示少量关键指标。
4. 文案：面向路人，避免工程调试词；调试信息保留在 /#/pose。
5. 响应式：优先考虑 16:9 横屏液晶屏，同时保证普通浏览器窗口不溢出。
6. 人体结构显示：继续基于 MediaPipe Pose 现有关键点，使用更粗的肢体线、半透明躯干面、关节节点和左右肢体颜色区分增强可读性。

## 边界

1. Task7 不负责新增动作识别算法。
2. Task7 不负责解决 SAM 3D Body、registration 或侧身算法问题。
3. Task7 不追求商业化完整活动页，只做演示版界面体验优化。
4. Task7 不在页面放复杂操作按钮，/#/demo 应尽量自动运行。
5. Task7 只做“显示更详细”，不做“识别更详细”；DensePose、人体分割、SAM 3D Body 或人体 mesh 等识别/模型升级放到 Task8。
