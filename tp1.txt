### Phase 4: 开合跳状态识别

- [x] 4.1 提取开合跳所需关键点
      → 肩膀
      → 手腕
      → 髋部
      → 脚踝
      → 关键点缺失时返回 unknown
      → 已完成: 使用左右肩、左右手腕、左右髋、左右脚踝
      → 已完成: 缺少必要关键点或肩宽过小时返回 unknown

- [x] 4.2 实现 open / closed 状态判断
      → 双脚间距大于肩宽阈值时，脚部为打开
      → 双手腕高于或接近肩部，且横向距离达到阈值时，手臂为打开
      → 手臂和脚部同时打开时，状态为 open
      → 双脚回到窄间距且手腕回落时，状态为 closed
      → 已完成: 脚部状态独立判断 open / closed / unknown
      → 已完成: 手臂状态独立判断 open / closed / unknown
      → 已完成: 脚部和手臂同时 open 时整体为 open
      → 已完成: 脚部和手臂同时 closed 时整体为 closed
      → 已完成: 其他组合返回 unknown
      → 待验证: 浏览器实测阈值是否适合当前摄像头距离和视角

- [x] 4.3 增加状态防抖
      → 连续若干帧满足条件后才切换状态
      → open / closed 使用不同阈值形成滞后区间
      → 抖动帧不触发计数
      → 已完成: 连续 3 帧相同 open / closed 后才切换稳定动作状态
      → 已完成: 脚部 open / closed 使用肩宽比例滞后阈值
      → 已完成: 手臂 open / closed 使用手腕距离和手腕高度滞后阈值
      → 已完成: 调试面板显示即时状态、稳定状态和防抖进度

- [ ] 4.4 验证站立状态
      → 用户站立不动时不持续误计数
      → 用户半动作或手脚不完整时可以不计数
      → 关键点丢失时不继续累加
      → 待验证: npm run dev 后用户实测站立、半动作和完整开合跳状态变化

### Phase 5: 开合跳计数和运动计时

- [ ] 5.1 实现计数状态机
      → 初始状态: waiting_closed
      → 检测到 closed 后进入 ready
      → ready 检测到 open 后进入 opened
      → opened 检测到 closed 后 count + 1，并回到 ready

- [ ] 5.2 实现开始、暂停、重置
      → 开始后才累计时间和次数
      → 暂停后停止累计
      → 重置后清空 count / elapsedMs / calories / 状态机

- [ ] 5.3 计算运动时长
      → 指标: elapsedMs
      → 暂停时间不计入运动时长
      → 页面展示为易读时间格式

- [ ] 5.4 计算每分钟频率
      → 公式: frequencyPerMin = count / elapsedMinutes
      → 页面展示保留 1 位小数
      → elapsedMinutes 为 0 时避免除零错误

### Phase 6: 开合跳 MET 和热量估算

- [ ] 6.1 实现开合跳体重输入
      → 默认值: 60 kg
      → 用户可修改
      → 非法输入需要回退或提示

- [ ] 6.2 实现开合跳三档 MET
      → 0 - 30 次/分钟: 6.0 MET
      → 30 - 50 次/分钟: 8.0 MET
      → 50 次/分钟以上: 10.0 MET
      → 频率变化后 MET 实时变化

- [ ] 6.3 实现热量估算公式
      → 公式: 总消耗 kcal = MET × 3.5 × 体重kg ÷ 200 × 运动分钟数
      → 修改体重后 kcal 实时更新
      → 暂停状态下 kcal 不继续增长

- [ ] 6.4 展示实时指标
      → 次数
      → 运动时长
      → 次/分钟
      → 当前 MET
      → 估算 kcal

### 开合跳置信度和验收

- [ ] 计算关键点完整率
      → 根据开合跳所需关键点判断
      → 缺失过多时置信度降低

- [ ] 判断全身入镜和居中情况
      → 脚踝、髋部、肩膀、手腕应尽量在画面内
      → 人体明显偏出画面时置信度降低

- [ ] 判断动作状态稳定性
      → open / closed 周期连续时置信度提高
      → 状态长期 unknown 或频繁抖动时置信度降低

- [ ] 开合跳实测
      → 明显完成 1 次开合跳时 count + 1
      → 连续开合跳时次数持续增加
      → 站立不动时不持续误计数
      → 关键点丢失时不继续错误累加
