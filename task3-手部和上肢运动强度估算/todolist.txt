# Task3 手部和上肢运动强度估算 — 实施清单

> 状态：进行中。  
> 目标：将 /#/gesture 从“挥手计数页”升级为“上肢运动强度估算页”。手部属于上肢的一部分，但 Task3 不应只依赖手掌轨迹；即使动作无法精确命名，也能根据手、腕、肘、肩等上肢关键点运动量，给出持续时间、强度档位、估算 kcal 和置信度。  
> 原则：继续以浏览器端前端为主；kcal 偏演示可解释；不追求动作名称识别准确率；不做商业级或医学级热量估算。  
> 关键判断：Task3 成功标准不是“识别出抬手、晃胳膊、伸懒腰等所有动作名”，而是“只要上肢明显运动，就能估算运动量和粗略消耗”。

## Phase 2: 上肢运动量特征

- [x] 2.1 新增上肢活动历史数据
      → 新增 upperBodyMotionHistory
      → 记录左/右肩、肘、腕、手部中心点、时间戳、即时速度
      → 用于计算运动量和持续时间
      → 已完成: GestureView 新增 upperBodyMotionHistory，接入左右肩肘腕和手部中心点

- [x] 2.2 计算上肢关键点位移量
      → 指标: 最近时间窗口内肩、肘、腕、手部中心点的位移
      → 输出: movementDistance / upperLimbJointDistance
      → 用途: 判断是否存在明显上肢活动
      → 已完成: 基于左右上肢关节链计算 movementDistance

- [x] 2.3 计算运动速度
      → 指标: 短窗口速度、平均速度、峰值速度
      → 输出: movementSpeed
      → 用途: 区分低 / 中 / 高强度活动
      → 已完成: 基于最近帧位移和时间差计算 movementSpeed

- [x] 2.4 计算运动幅度
      → 指标: 手腕/肘部/手部中心点的 xRange、yRange、相对肩部幅度
      → 输出: movementAmplitude
      → 用途: 区分小幅手指 / 手腕动作与大幅上肢动作
      → 已完成: 基于最近窗口内左右肘、腕、手部中心点轨迹计算 movementAmplitude

- [x] 2.5 计算活动持续时间
      → 只在检测到明显运动量时累计 activeUpperBodyMs
      → 手部静止、未检测到手部时不继续累计有效活动时间
      → 与普通记录时长区分
      → 已完成: movementDistance 达到阈值时累计 activeUpperBodyMs

## Phase 3: 上肢活动强度分档

- [ ] 3.1 新增上肢活动分数
      → 输入: movementDistance、movementSpeed、movementAmplitude、activeUpperBodyMs
      → 输出: upperBodyActivityScore，范围 0 - 100
      → 说明: 该分数不等于动作识别概率

- [ ] 3.2 新增强度档位
      → 档位: 静止 / 低强度 / 中强度 / 高强度
      → 依据: 上肢活动分数、速度和幅度
      → 要求: 分档应可解释，便于工位测试调整阈值

- [ ] 3.3 保留已知动作辅助标签
      → 挥手、抬胳膊、手向前、手向后、出拳等仍可作为辅助标签
      → 抬胳膊作为基本动作保存，可服务后续开合跳、上肢训练或全身动作识别
      → 未知动作时也能输出强度档位
      → 动作名不作为 kcal 唯一入口

## Phase 4: 上肢活动 MET 和 kcal

- [ ] 4.1 新增上肢活动 MET 档位
      → 说明: Task3 kcal 偏演示可解释
      → 建议初始档位:

        - 低强度上肢活动: 1.5 MET
        - 中强度上肢活动: 2.2 MET

      → 后续可根据测试调整阈值和数值

- [ ] 4.2 计算上肢活动 kcal
      → 公式: upperBodyKcal = upperBodyMet × 3.5 × weightKg ÷ 200 × activeUpperBodyMinutes
      → 与 waveCalories 区分

- [ ] 4.3 保留挥手 kcal 作为动作示例
      → waveCalories 继续存在

      → 页面文案说明二者含义不同

## Phase 5: 置信度和解释

- [ ] 5.1 新增上肢活动置信度
      → 等级: 高 / 中 / 低
      → 输入: 手部可见率、肩肘腕关键点完整率、采样帧数、运动稳定性、活动持续时间
      → 输出: upperBodyConfidence

- [ ] 5.2 新增置信度原因
      → 示例:
        - 手部持续入镜
        - 肩部 / 肘部参考点稳定
        - 活动时间较短
        - 运动幅度较小
        - 摄像头残影或信号不稳定
      → 页面展示 1-3 条主要原因

- [ ] 5.3 区分“动作置信度”和“热量估算置信度”
      → 动作标签可以不确定
      → 只要运动量稳定，热量估算仍可有中等置信度
      → 避免用户误解为“没识别出动作名就不能估算”

### kcal 定位

Task3 继续保持 kcal “演示可解释”。重点展示运动时间、频率、幅度、速度和置信度如何影响估算结果；贴近真实代谢消耗放到后续个体化和多模态阶段。
