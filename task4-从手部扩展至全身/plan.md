# Task4：从手部扩展至全身

## 本 task 要完成的内容

将系统从手部 / 上肢运动强度估算扩展到全身运动强度估算。恢复并承接原先从 Task1 剪切出来的开合跳内容，同时可以加入高抬腿、深蹲等全身动作示例。

本 task 的目标不是只做全身动作分类，而是建立全身关键点的运动量、周期、幅度、频率和基础动作质量反馈。

## 预期结果

1. `/pose` 成为全身动作和全身运动强度估算入口。
2. 系统能检测人体关键点并绘制全身骨架。
3. 能对开合跳等全身动作做周期计数、频率统计和 MET / kcal 演示估算。
4. 能输出全身关键点完整率、全身入镜情况、动作周期稳定性和估算置信度。
5. 对部分动作给出基础质量反馈，例如幅度不足、节奏不稳定、关键点缺失。

## 可能的实现方案和技术栈

1. 前端：Vite + Vue 3 + TypeScript。
2. 视觉识别：MediaPipe Pose Landmarker。
3. 绘制：Canvas 全身骨架和关键点。
4. 算法：规则算法 + 状态机 + 关键点轨迹统计。
5. 动作示例：优先开合跳，其次高抬腿、深蹲。
6. 热量估算：动作基础 MET + 频率系数 + 幅度系数，仍以演示可解释为主。

## 边界

1. 不追求覆盖所有健身动作。
2. 不支持多人同时入镜。
3. 不做负重、平板支撑、靠墙静蹲等视觉盲区较明显动作的精确估算。
4. 不急于引入复杂动作识别模型；模型化方案放到后续算法优化。
