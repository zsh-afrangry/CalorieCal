# Task3：手部和上肢运动强度估算

## 本 task 要完成的内容

将 Task1/Task2 的“挥手示例”扩展为更通用的上肢运动强度估算。手部属于上肢的一部分，但 Task3 不应只依赖手掌轨迹；后续需要沿着手、腕、肘、肩等关节连接处扩展。目标不是识别尽可能多的手势名称，而是即使动作无法精确命名，也能根据上肢关键点运动量给出粗略强度、持续时间、频率、估算 kcal 和置信度。

示例动作可以包括挥手、抬胳膊、晃胳膊、伸懒腰、手臂前后移动、简单挥动等。其中抬胳膊、手向前移动、手向后移动等可以作为“基本动作”保存，因为它们可能是出拳、开合跳、上肢训练或其他动作的组成部分。

基本动作和组合动作需要区分：基本动作是可复用的单一运动片段，例如手向前移动、手向后移动、左抬胳膊、右抬胳膊；组合动作由多个基本动作、姿态关系和时序规则组成，例如出拳、开合跳。出拳不是单纯“手向前移动”，开合跳也不是单纯“两个胳膊向上抬”，它们都需要组合判断。

## 预期结果

1. `/gesture` 从“挥手计数页”升级为“上肢活动强度估算页”。
2. 系统能统计手掌、手腕、肘部、肩部相关关键点的位移、速度、幅度和运动持续时间。
3. 已知基本动作可以显示辅助标签，例如抬胳膊、手向前移动、手向后移动；组合动作可以后续单独判断，未知动作也能按低 / 中 / 高强度给出粗略估算。
4. kcal 仍保持演示可解释，优先说明“基于上肢运动量粗略估算”。
5. 置信度应考虑关键点可见率、运动稳定性、画面残影和是否持续入镜。

## 可能的实现方案和技术栈

1. 前端：Vite + Vue 3 + TypeScript。
2. 视觉识别：MediaPipe Hand Landmarker + Pose Landmarker。Hand Landmarker 负责手部细节，Pose Landmarker 提供肩部、肘部、手腕和躯干参考点。
3. 算法：基于上肢关节链关键点轨迹的运动量积分、速度分档、幅度分档和时间窗口统计。
4. 热量估算：从“挥手 MET 档位”扩展为“上肢活动 MET 档位”，先用规则表演示。
5. 后端：暂不强制引入；如果规则表、参数配置或记录管理变复杂，再考虑 FastAPI。

## 边界

1. 不追求动作名称识别准确率，但允许保存常见基本动作作为辅助标签，并明确区分基本动作和组合动作。
2. 不做商业级 kcal 精准估算。
3. 不要求识别静态发力或负重强度。
4. 不做全身动作质量纠错，全身能力进入 Task4。
