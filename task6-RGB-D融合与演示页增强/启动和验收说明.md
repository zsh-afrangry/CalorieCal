# Task6 启动和验收说明

## 启动流程

1. 启动 RGB-D depth server

```powershell
python "task5-SAM 3D Body尝试\openni_depth_server.py" --openni-redist "OpenNI_2.3.0.86"
```

2. 检查 depth server

```powershell
Invoke-RestMethod "http://127.0.0.1:8765/health"
Invoke-RestMethod "http://127.0.0.1:8765/depth?normX=0.5&normY=0.5"
```

3. 启动前端

```powershell
cd frontend
npm run dev
```

4. 打开页面

```text
http://localhost:5173/#/pose
http://localhost:5173/#/demo
```

## 工位验收清单

1. 正对摄像头时，/#/demo 显示 RGB-D 已接入。
2. 正对摄像头手前伸时，前后方向区域显示手前移证据。
3. 正对摄像头手收回时，前后方向区域显示手后移或接近平面。
4. 关闭 depth server 后，/#/demo 不崩溃，并显示 RGB-D 未连接 / 使用 2D 估算。
5. 侧身或躯干参考不稳定时，/#/demo 出现请正对摄像头或躯干参考不稳定类提示。
6. depth 采样疑似背景时，/#/demo 出现 depth 采样疑似背景提示。
7. /#/pose 保留调试细节；/#/demo 不展示过多调试噪音。

## 当前限制

1. registration 开启失败仍未解决，当前 /#/demo 使用 raw depth 融合。
2. 侧身和遮挡只能降置信度并提示，不能完全解决。
3. RGB-D depth 只参与前后方向证据和估算置信度，不直接覆盖动作名称。
4. kcal 仍是演示可解释口径，不代表真实代谢测量。
