# Task1：最小 MVP

## 本 task 要完成的内容

以 `/#/gesture` 的“挥手”为示例，跑通一个可运行的浏览器端最小链路：

```text
摄像头 -> 手部关键点 -> 挥手周期计数 -> 频率 -> MET -> kcal -> 基础置信度
```

本 task 不追求热量精准，也不追求覆盖所有动作。kcal 更偏“演示可解释”，用于证明视觉关键点、动作周期、频率和热量估算可以连成一条真实可运行的链路。

## 预期结果

1. 用户打开前端后可授权摄像头并看到实时画面。
2. 页面可检测手部关键点并绘制骨架。
3. 慢速或中速挥手时可以累计挥手次数。
4. 页面展示挥手时长、频率、MET、估算 kcal 和基础置信度。
5. 摄像头、模型加载和未检测到手部时有基础错误提示。

## 可能的实现方案和技术栈

1. 前端：Vite + Vue 3 + TypeScript。
2. 视觉识别：MediaPipe Tasks Vision 的 Hand Landmarker。
3. 绘制：Canvas。
4. 算法：基于手部中心点、横向位移、换向次数和时间窗口的启发式规则。
5. 热量估算：使用 `kcal = MET * 3.5 * weightKg / 200 * minutes`，MET 先使用挥手频率的演示档位。

## 边界

1. 不做后端、数据库、登录和历史记录。
2. 不训练模型。
3. 不做医学级或商业级 kcal 精准估算。
4. 原开合跳和全身动作内容迁移到后续 task。
