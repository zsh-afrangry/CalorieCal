# Task8：更详细人体识别方案研究

## 本 task 要完成的内容

Task8 负责研究“识别更详细”的方案，用于补足 MediaPipe Pose 只有 33 个关键点、躯干和四肢只能粗略表示的问题。它与 Task7 分工不同：Task7 只优化 /#/demo 的显示效果，Task8 才评估人体分割、DensePose、SAM 3D Body、人体 mesh、RGB-D 点云融合等更细粒度人体识别或建模方案。

## 预期结果

1. 明确当前 MediaPipe Pose 的上限：关键点足够做运动量估算，但不提供大腿、小腿、躯干表面的细粒度结构。
2. 评估 MediaPipe segmentation mask 是否能用于人体轮廓和展示增强。
3. 评估 DensePose 是否适合细分人体表面区域，例如躯干、大腿、小腿、手臂。
4. 评估 SAM 3D Body / Human Mesh Recovery 是否适合解决侧身、遮挡、躯干参考面不稳定。
5. 评估 RGB-D depth 是否能结合人体分割或 mesh 形成更可靠的身体表面/空间幅度估算。
6. 给出是否值得接入后端、离线实验或实时演示链路的判断。

## 可能的实现方案和技术栈

1. 继续保留 MediaPipe Pose / Hand 作为实时主链路。
2. Python 后端评估 DensePose、人体分割或 3D mesh 模型。
3. RGB-D 方案继续使用 OpenNI depth server，必要时增加深度图采样、人体区域过滤或点云实验。
4. SAM 3D Body 先做离线对比，不直接进入实时 MVP。
5. 若模型过重，则只作为论文展示、离线分析或后续产品化方向。

## 边界

1. Task8 不负责 /#/demo 的视觉布局和大屏 UI，这些属于 Task7。
2. Task8 不急于接入实时页面，先做方案研究和小实验。
3. Task8 不以“模型更高级”为目标，必须服务运动时间、频率、幅度、强度、估算消耗和置信度。
4. 若新模型不能明显改善侧身、遮挡、身体参考面或空间幅度估算，则暂缓接入。
